## **🎯 What is This Project?**

This is an **Unified Executive Dashboard** system that collects, organizes, and serves business data from multiple sources so executives can make informed decisions.

---

## **📋 The Problem It Solves**

**Current Challenge:**
- Business data is scattered across different systems (Salesforce, Excel spreadsheets, various tools)
- Executives need to manually gather data from multiple places
- Data quality varies, with duplicates and inconsistencies
- Historical tracking is difficult - "What did this look like last month?"
- No single source of truth for KPIs (Key Performance Indicators)

**What This System Does:**
Automatically pulls data from all sources, cleans it, organizes it, and presents it through a unified API for dashboards.

---

## **🎁 Key Benefits**

1. **Single Source of Truth** - All business metrics in one place
2. **Automated Data Collection** - No manual exports needed
3. **Time Travel** - See what data looked like at any point in history
4. **Data Quality** - Automatic cleaning and deduplication
5. **Privacy Ready** - Built-in data masking for sensitive information
6. **Easy to Extend** - Add new data sources just by creating a YAML config file

---

## **🏗️ Technical Architecture (Simple Explanation)**

Think of it as a **three-story building**:

### **🏪 Ground Floor (RAW ZONE)** - "The Loading Dock"
- **What happens:** Data comes in from 7 sources:
  - **Salesforce** (4 datasets): Accounts, Contacts, Opportunities, Activities
  - **Excel files** (3 datasets): Projects, Resource Allocations, Employee Skills
- **How:** Python "collectors" automatically fetch/read this data
- **Storage:** Everything stored in PostgreSQL with full history (bi-temporal tracking)
- **Special feature:** Every change is tracked - you can see what the data looked like yesterday, last week, or any time

### **🏢 Middle Floor (REFINED ZONE)** - "The Quality Control Room"
- **What happens:** Raw data gets cleaned and standardized
- **Process:** dbt (data build tool) removes duplicates, fixes data types, standardizes formats
- **Output:** Clean, consistent tables ready for analysis (accounts, contacts, projects, employees, etc.)

### **🏛️ Top Floor (TRUSTED ZONE)** - "The Executive Summary Room"
- **What happens:** Data gets aggregated into business KPIs
- **Examples:**
  - Account KPIs (pipeline value, active projects, NPS scores)
  - Delivery status (project health metrics)
  - Team allocation (who's working on what)
  - Pipeline summary (sales forecasts)
  - Service line performance
- **Output:** Ready-to-display metrics for executive dashboards

### **📡 The Roof (API LAYER)** - "The Service Window"
- **FastAPI** serves the trusted zone data to frontend dashboards
- RESTful endpoints with automatic documentation
- Health checks and monitoring built-in

---

## **🔄 Technical Flow (Step-by-Step)**

```
1. RUN PIPELINE
   ↓
2. COLLECTORS fetch data
   ├─ REST Collector → Salesforce API (with OAuth)
   └─ File Collector → Excel files from landing folders
   ↓
3. DATA LANDS in raw_zone tables (PostgreSQL)
   - Each row tagged with timestamps (when loaded, when valid)
   - Old versions kept (bi-temporal = transaction time + valid time)
   ↓
4. DBT TRANSFORMS run automatically
   ├─ refined_zone: Clean & deduplicate
   └─ trusted_zone: Aggregate KPIs
   ↓
5. FASTAPI serves the data
   └─ Dashboard queries → GET /api/v1/account-kpis
   ↓
6. EXECUTIVES see up-to-date metrics
```

- **SQLAlchemy** - Database ORM
- **Flyway** - Database migration management

---

## **💡 Smart Design Choices**

1. **Template Method Pattern** - All collectors inherit from base class, override only what's unique
2. **YAML Configuration** - Add new data sources without writing Python code
3. **Bi-temporal Storage** - Track both "when data changed" and "when change was recorded"
4. **Watermark Cursor** - Only fetch new/changed data (efficient)
5. **Data Masking** - Pseudonymize sensitive fields while keeping joins working

## **🚀 Usage Example**
