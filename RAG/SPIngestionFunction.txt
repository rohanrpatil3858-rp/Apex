## 🎯 **WHAT THIS PROJECT DOES**

This is an **automated SharePoint document indexing system** that:
- **Monitors** SharePoint for new/changed/deleted files
- **Downloads** and processes those files
- **Converts** them into searchable data with AI embeddings
- **Stores** everything in Azure AI Search for smart searching

Think of it like a **robot librarian** that constantly watches a folder, reads new books, understands their content, and creates a smart catalog so you can search and find information easily.

---

## 🏗️ **PROJECT ARCHITECTURE**

### **Main Components (8 Files):**

#### **1. Program.cs** - The Starting Point
- Sets up the application when it starts
- Loads configuration from local.settings.json
- Registers all services (Graph, Embedding, Search, etc.)
- Think of it as the **power button** that turns everything on

#### **2. IngestionFunction.cs** - The Main Worker (The Brain)
**Runs every 1 minute automatically**

The main flow:
```
1. Check what changed in SharePoint (using delta token)
2. For each changed file:
   - If deleted → Remove from search index
   - If new/updated:
     a. Download the file content
     b. Break it into chunks
     c. Create AI embeddings for each chunk
     d. Upload to Azure AI Search
3. Save checkpoint (delta token) for next run
```

---

### **Supporting Services (5 Helper Files):**

#### **3. GraphService.cs** - SharePoint Connector
**What it does:**
- Connects to SharePoint using Microsoft Graph API
- Authenticates with Azure AD (gets permission token)
- Gets list of changed files (delta query)
- Downloads file content

**Key Methods:**
- `GetToken()` - Gets permission to access SharePoint
- `GetDelta()` - Finds what files changed since last check
- `DownloadFile()` - Downloads a specific file's content

#### **4. EmbeddingService.cs** - AI Understanding
**What it does:**
- Takes text and converts it to **vector embeddings** (numbers that represent meaning)
- Uses Azure OpenAI API (text-embedding model)
- These embeddings allow semantic search (searching by meaning, not just keywords)

**Example:** "How to reset password?" and "Password recovery steps" would have similar embeddings even though words are different.

#### **5. SearchService.cs** - Azure AI Search Manager
**What it does:**
- Uploads documents to Azure AI Search index
- Deletes documents when files are removed
- Uses the search index name from configuration

**Key Methods:**
- `UploadDocuments()` - Adds/updates search documents
- `DeleteByFileId()` - Removes all chunks of a deleted file

#### **6. ChunkingService.cs** - Text Splitter
**What it does:**
- Breaks large documents into smaller chunks (default: 1000 characters)
- Adds overlap between chunks (150 characters) so context isn't lost
- Why? AI embedding models have size limits, and smaller chunks give better search results

**Example:** A 5000-character document becomes 5-6 overlapping chunks.

#### **7. DeltaStateStore.cs** - Memory Keeper
**What it does:**
- Saves a "checkpoint" (delta token) to a local file
- Remembers what was processed last time
- Prevents reprocessing the same files over and over

**Think of it as:** A bookmark that remembers where the robot librarian left off.

#### **8. SearchDocumentModel.cs** - Data Blueprint
**What it does:**
- Defines the structure of documents stored in Azure AI Search

**Fields:**
- `id` - Unique identifier (fileId_chunkNumber)
- `title` - File name
- `content` - The actual text chunk
- `contentVector` - AI embedding (array of numbers)
- `createdDate` - When file was created
- `folderName` - Where file is located
- `author` - Who created it

## 🔄 **TECHNICAL FLOW (Step-by-Step)**

```
⏰ Timer triggers every 1 minute
    ↓
1️⃣ READ deltaToken.txt (checkpoint from last run)
    ↓
2️⃣ CALL Microsoft Graph API with delta token
    → Get list of changed files since last run
    ↓
3️⃣ FOR EACH changed file:
    
    🗑️ IF file is DELETED:
       → Search and remove all chunks from Azure AI Search
    
    📄 IF file is NEW or UPDATED:
       ↓
       a) DOWNLOAD file content from SharePoint
       ↓
       b) CHUNK the text (1000 chars, 150 overlap)
       ↓
       c) FOR EACH chunk:
          → Generate AI embedding using Azure OpenAI
          → Create SearchDocument with:
             - id (fileId_chunkIndex)
             - content (chunk text)
             - vector (embedding)
             - metadata (title, author, etc.)
       ↓
       d) UPLOAD all chunks to Azure AI Search
    ↓
4️⃣ SAVE new delta token to deltaToken.txt
    ↓
5️⃣ DONE - Wait for next trigger
```

## 🎓 **SIMPLE SUMMARY**

**What:** An automated system that keeps Azure AI Search in sync with SharePoint files.

**How:** 
1. Timer runs every minute
2. Checks SharePoint for changes
3. Downloads new/updated files
4. Breaks them into chunks
5. Creates AI embeddings for smart search
6. Uploads to Azure AI Search
7. Deletes removed files
8. Saves checkpoint and repeats

**Why:** So users can search SharePoint documents using AI-powered semantic search (search by meaning, not just keywords), with results coming from Azure AI Search instead of regular SharePoint search.

**Technology:** C# Azure Function + Microsoft Graph + Azure OpenAI + Azure AI Search

**Real-world use:** Corporate knowledge base, document discovery, intelligent search over company files.
